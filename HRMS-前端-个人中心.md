# 2. 详细设计

## 2.1 前端迭代目标

1. 个人中心首页：左侧菜单 + 右侧内容区布局，5个功能模块
2. 我的档案：信息展示（敏感字段脱敏）+ 可编辑字段修改 + 锁定字段提示
3. 我的考勤：日历视图 + 打卡按钮 + 月度统计
4. 我的请假：请假列表 + 请假表单 + 审批进度 + 假期余额
5. 我的薪资：工资条列表 + 详情（二次验证）+ 趋势图
6. 账号安全：修改密码 + 手机绑定 + 登录日志

### 2.2.2 我的档案

##### UI

- 分区卡片：基础信息 / 个人信息 / 工作信息 / 薪资与合同信息
- 每张卡片右上角「编辑」按钮（仅当有 editableFields 时显示）
- 编辑模式：可编辑字段变表单控件，锁定字段 disabled + Lock Icon + Tooltip 提示

##### 档案字段展示

与员工档案详情页（HR视角）基本一致，但：
- **仅展示自己**的数据（不用员工选择器）
- 敏感字段脱敏规则相同：
  - 手机号：138****1234
  - 身份证号：3301**********1234
  - 银行卡号：****8899

##### 锁定字段提示

| 锁定字段 | Tooltip 文案 |
| --- | --- |
| 手机号 | 如需修改手机号请联系HR，需走信息变更申请 |
| 所属部门 | 部门变更需走调岗流程 |
| 职位 | 职位变更需走调岗流程 |
| 职级 | 职级变更需走调岗流程 |
| 薪资信息 | 薪资调整请联系HR |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /api/v1/my/profile | 查看我的档案（含fieldPermissions） |
| PUT /api/v1/my/profile | 编辑我的档案 |

### 2.2.3 我的考勤

##### UI

- **顶部打卡区**（工作日显示）：
  - 当前日期 + 实时时钟
  - 上班打卡按钮 / 下班打卡按钮（状态互斥）
  - 已打卡显示时间 + 判定结果（正常/迟到/早退）

- **月度统计卡片**：
  - 应出勤 / 实际出勤 / 迟到次数 / 早退次数 / 请假天数 / 年假余额

- **考勤日历**：
  - AntD Calendar 组件
  - 日期格子颜色标记：绿=正常 / 橙=迟到早退 / 红=旷工 / 蓝=请假 / 灰=缺卡
  - 点击日期 Popover 显示详细打卡记录

- **快捷操作**：
  - 「申请请假」按钮 → 跳转我的请假
  - 「申请补卡」按钮 → 跳转补卡表单（针对缺卡日期）

##### 打卡交互

```typescript
// 上班打卡
onClick: () => {
  POST /api/v1/my/attendance/clock-in
  // 成功 → 按钮变灰 + 显示打卡时间 + Toast "打卡成功·正常"
  // 迟到 → Toast 警告 "迟到打卡，已记录"
  // 不在打卡范围 → Toast 错误 "当前位置/网络不允许打卡"
}
```

##### 所需API

| 接口 | 说明 |
| --- | --- |
| POST /api/v1/my/attendance/clock-in | 上班打卡 |
| POST /api/v1/my/attendance/clock-out | 下班打卡 |
| GET /api/v1/my/attendance/calendar?month= | 考勤日历 |
| GET /api/v1/my/attendance/statistics?month= | 月度统计 |

### 2.2.4 我的请假

##### UI

- **假期余额卡片**：年假余额 + 调休余额（醒目展示）
- **请假列表**：Tab 筛选（全部/草稿/审批中/已通过/已拒绝/已取消）
- **「申请请假」按钮** → 跳转请假表单页（或 Modal）

##### 请假详情

- 点击列表项展开/跳转详情
- 底部显示**审批进度时间线**

##### 取消操作

- 仅审批中状态可取消
- 二次确认弹窗

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /api/v1/my/leave-balance | 假期余额 |
| GET /api/v1/my/leave-applications | 请假列表 |
| GET /api/v1/my/leave-applications/{id} | 请假详情（含审批进度） |
| POST /api/v1/my/leave-applications/{id}/cancel | 取消申请 |

### 2.2.5 我的薪资

##### UI

- **工资条列表**：
  - 按月倒序，仅显示已通过/已发放批次
  - 每项：月份 + 应发 + 实发 + 状态 Tag
  - 可查看项可点击，不可查看项灰色

- **工资条详情 Modal**：
  - 首次查看 → 二次验证（短信验证码 / 密码）
  - 验证通过 → 展示仿真实工资条样式
  - 收入区 + 扣除区 + 实发金额

- **薪资趋势图**：
  - AntV 折线图：近6个月实发工资变化
  - 数据源：GET /api/v1/my/salary-trend?months=6

##### 二次验证弹窗

| 字段 | 说明 |
| --- | --- |
| 验证方式 | Radio：短信验证码 / 登录密码 |
| 短信验证码 | 验证方式=短信时显示 + 发送按钮 + 60s倒计时 |
| 密码 | 验证方式=密码时显示 Password Input |
| 确认 | 提交验证 |

- 连续3次失败 → 锁定30分钟提示
- 验证通过 → 10分钟内免验证

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /api/v1/my/payslips | 工资条列表 |
| POST /api/v1/my/payslips/{batchId}/verify | 二次验证 |
| GET /api/v1/my/payslips/{batchId} | 工资条详情 |
| GET /api/v1/my/salary-trend?months=6 | 薪资趋势 |

### 2.2.6 账号安全

##### 修改密码

| 字段名称 | 说明 | 输入方式 | 是否必填 | 校验规则 |
| --- | --- | --- | --- | --- |
| 旧密码 | 当前密码 | Password Input | Y | - |
| 新密码 | - | Password Input | Y | >=8位 + 大小写 + 数字 |
| 确认新密码 | - | Password Input | Y | 与新密码一致 |

- 提交成功后 → Toast 提示 → 强制跳转登录页（所有设备重新登录）
- 错误提示：
  - "旧密码错误"
  - "新密码不能与旧密码相同"
  - "不能使用近期使用过的密码"
  - "密码需至少8位，包含大小写字母和数字"

##### 绑定/修改手机号

| 字段名称 | 说明 | 输入方式 | 是否必填 |
| --- | --- | --- | --- |
| 新手机号 | - | Input | Y |
| 短信验证码 | 发送到新手机号 | Input + SendButton(60s倒计时) | Y |

##### 登录日志

- 表格分页展示：登录时间 + IP + 设备(User-Agent解析) + 登录方式 + 是否成功
- 仅支持查看，不支持删除

##### 所需API

| 接口 | 说明 |
| --- | --- |
| PUT /api/v1/my/password | 修改密码 |
| POST /api/v1/my/phone/send-code | 发送短信验证码 |
| PUT /api/v1/my/phone | 绑定/修改手机号 |
| GET /api/v1/my/login-logs | 登录日志 |

# 5. 其他

## 5.1 风险评估

- 个人中心聚合多个模块，需确保各子模块数据一致性
- 二次验证的短信发送有频率限制和成本
- 密码修改后强制登出，需处理好前端路由跳转和状态清理
- 工资条数据敏感，需确保前端不缓存到 localStorage
