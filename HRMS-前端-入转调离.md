# 2. 详细设计

## 2.1 前端迭代目标

1. 入职管理：候选人列表（含状态筛选） + 入职表单（草稿/提交） + 入职确认
2. 转正管理：待转正提醒列表 + 转正评估表单 + 审批结果处理
3. 调岗管理：调岗申请表单（原部门→新部门） + 调岗历史查看
4. 离职管理：离职申请表单 + 交接人指定 + 离职生效确认

## 2.2 迭代具体描述

### 2.2.1 入职管理列表页

##### UI&交互

- 顶部 Tab 状态筛选：全部 / 草稿 / 审批中 / 已批准待入职 / 已入职 / 已拒绝 / 已放弃
- 表格列：候选人姓名、手机号、申请部门、申请职位、预计入职日期、状态（Tag+颜色）、创建时间、操作
- 「新建入职」按钮 → 跳转入职表单页

##### 列表搜索字段

| 字段名称 | 说明 | 输入方式 | 是否必填 | 字段类型 | 数据源 |
| --- | --- | --- | --- | --- | --- |
| 关键词 | 姓名/手机号 | Input | N | string | - |
| 状态 | Tab切换 | - | N | number | 枚举 |
| 申请日期 | 日期范围 | DatePicker | N | {start,end} | - |

###### 操作按钮

| 按钮 | 交互 | 显示控制 |
| --- | --- | --- |
| 查看详情 | 进入入职详情页（含审批进度） | 始终 |
| 编辑 | 进入编辑表单（复用入职表单） | 状态=草稿 或 已拒绝 |
| 删除 | 删除草稿 | 状态=草稿，二次确认 |
| 确认入职 | 打开确认弹窗（填写实际入职日期） | 状态=已批准待入职 |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /api/v1/onboarding-applications | 入职申请列表 |
| POST /api/v1/onboarding-applications | 新建入职申请（保存草稿/提交审批） |
| PUT /api/v1/onboarding-applications/{id}/confirm-hire | 确认入职 |
| PUT /api/v1/onboarding-applications/{id}/abandon | 标记放弃 |

### 2.2.2 入职申请表单页

##### 表单字段

| 字段名称 | 说明 | 输入方式 | 是否必填 | 字段类型 | 数据源 |
| --- | --- | --- | --- | --- | --- |
| 姓名 | - | Input | Y | string | - |
| 性别 | - | Radio | Y | number | 枚举：男/女 |
| 手机号 | - | Input | Y | string | 正则校验 |
| 邮箱 | - | Input | Y | string | 邮箱校验 |
| 身份证号 | - | Input | Y | string | 正则校验 |
| 预计入职日期 | >= 今天 | DatePicker | Y | date | - |
| 所属部门 | - | TreeSelect | Y | number | GET /api/v1/departments/tree |
| 职位 | 选部门后联动过滤 | Select | Y | number | GET /api/v1/positions?departmentId= |
| 录用类型 | 全职/兼职/实习 | Select | Y | number | 枚举 |
| 试用期(月) | 默认取职位配置 | InputNumber | Y | number | 职位.defaultProbationMonths |
| 试用期薪资比例 | 80%-100% | Slider + InputNumber | Y | number | 0.8 |
| 直接汇报人 | 默认部门负责人 | Select | N | number | 搜索员工 |

###### 操作按钮

| 按钮 | 交互 | 二次确认 |
| --- | --- | --- |
| 保存草稿 | 调用 POST，submit=false | N |
| 提交审批 | 调用 POST，submit=true | Y，"提交后将进入审批流程" |
| 取消 | 返回列表 | Y，"未保存的更改将丢失" |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| POST /api/v1/onboarding-applications | 创建入职申请（草稿/提交） |
| POST /api/v1/onboarding-applications/{id}/submit | 提交审批（从草稿状态） |

### 2.2.3 转正管理

##### UI

- 页面顶部：待处理提醒区域（Alert 组件，显示"X 名员工试用期即将到期"）
- 列表展示待转正/转正中/已转正员工
- 点击"发起转正"→ 弹窗/新页填写评估表单

##### 转正评估表单

| 字段名称 | 说明 | 输入方式 | 是否必填 | 字段类型 | 数据源 |
| --- | --- | --- | --- | --- | --- |
| 员工信息 | 自动带出 | 只读展示 | - | - | - |
| 试用期起止日期 | 自动带出 | 只读展示 | - | date | - |
| 试用期表现评价 | 文本 | TextArea | Y | string | - |
| 审批结果 | 通过/延长试用/不通过 | Radio | Y (提交时) | number | 枚举 |
| 延长至日期 | 审批结果=延长时出现 | DatePicker | 条件必填 | date | - |
| 转正后薪资调整 | 选填 | InputNumber | N | number | - |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /api/v1/regularization-applications/pending | 待转正员工列表 |
| POST /api/v1/regularization-applications | 创建转正申请 |

### 2.2.4 调岗管理

##### 调岗申请表单

| 字段名称 | 说明 | 输入方式 | 是否必填 | 字段类型 | 数据源 |
| --- | --- | --- | --- | --- | --- |
| 员工 | 搜索选择 | Select + 搜索 | Y | number | GET /api/v1/employees |
| 当前部门/职位/职级 | 自动带出 | 只读展示 | - | - | - |
| 新部门 | 必须与当前不同 | TreeSelect | Y | number | 部门树 |
| 新职位 | 可选 | Select | N | number | 按新部门过滤 |
| 新职级 | 可选 | Cascader | N | string | 序列对照 |
| 新汇报人 | 可选 | Select | N | number | 员工搜索 |
| 薪资调整 | 可选（有值则触发额外审批） | InputNumber | N | number | - |
| 调岗原因 | 必填 | TextArea | Y | string | - |

##### 调岗历史

- 在员工详情页「调岗历史」Tab / 抽屉展示
- Timeline 展示：调岗日期 + 原岗位 → 新岗位
- 数据源：GET /api/v1/employees/{id}/change-history

##### 所需API

| 接口 | 说明 |
| --- | --- |
| POST /api/v1/transfer-applications | 创建调岗申请 |

## 5.1 风险评估

- 状态流转复杂，需确保前端状态枚举与后端一致
- 表单字段联动较多（部门→职位、职位→试用期），需处理好依赖关系
- 入职自动处理（工号生成+账号创建）后需在前端展示结果
