# 2. 详细设计

## 2.1 前端迭代目标

1. 员工列表页：默认展示字段 + 高级搜索 + 列表操作（查看/编辑/调岗/离职）
2. 员工档案详情页：分区展示四类信息（基础/个人/工作/薪资合同），按字段权限控制可见性和可编辑性
3. 员工档案编辑页：按角色动态渲染可编辑字段，锁定字段置灰提示
4. 变更历史：档案变更记录时间线展示

## 2.2 迭代具体描述

### 2.2.1 员工列表页

##### UI&交互

- 顶部高级搜索区（默认收起，点击展开）
- 表格展示，支持分页、排序
- 点击姓名进入档案详情页
- 每行操作列：查看详情 / 编辑 / 更多（调岗、离职）

##### 前端逻辑

###### 列表搜索字段

| 字段名称 | 说明 | 输入方式 | 是否必填 | 默认值 | 字段类型 | 提示文案 | 数据源 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 关键词 | 姓名/工号/手机号模糊匹配 | Input | N | - | string | 请输入姓名/工号/手机号 | - |
| 部门 | 部门树多选 | TreeSelect multiple | N | - | number[] | 请选择部门 | GET /api/v1/departments/tree |
| 职位 | 下拉多选 | Select multiple + 搜索 | N | - | number[] | 请选择职位 | GET /api/v1/positions |
| 在职状态 | 多选 | Select multiple | N | - | number[] | 请选择状态 | 枚举：试用期/正式/待离职/已离职 |
| 职级 | 多选 | Select multiple | N | - | string[] | 请选择职级 | GET /api/v1/positions/sequences |
| 入职日期 | 日期范围 | DatePicker.RangePicker | N | - | {start,end} | - | - |

###### 列表工具栏

| 按钮 | 交互 | 显示控制 |
| --- | --- | --- |
| 批量导出 | 导出当前筛选结果（Excel） | hr / system_admin |
| 高级搜索 | 展开/收起搜索区域 | 始终显示 |
| 重置 | 清空所有搜索条件 | 始终显示 |

###### 列表展示字段（默认）

| 字段名称 | 说明 | 默认值 | 宽度 |
| --- | --- | --- | --- |
| 姓名 | 点击跳转详情页 | N | 120 |
| 工号 | - | N | 120 |
| 部门 | - | N | 120 |
| 职位 | - | N | 140 |
| 职级 | - | N | 80 |
| 在职状态 | Tag 组件 + 颜色：试用期(蓝)/正式(绿)/待离职(橙)/已离职(灰) | N | 100 |
| 入职日期 | format YYYY-MM-DD | N | 120 |
| 操作 | 查看 / 编辑 / 更多 | N | 180 |

###### 操作按钮

| 按钮 | 交互 | 二次确认 | 显示/禁用控制 |
| --- | --- | --- | --- |
| 查看详情 | 跳转档案详情页 | N | 始终 |
| 编辑 | 跳转编辑页 | N | hr可编辑所有，部门主管部分字段，普通员工隐藏 |
| 调岗 | 跳转调岗申请页 | N | 在职状态为"试用期"或"正式"，且角色为hr |
| 离职 | 跳转离职申请页 | N | 在职状态为"试用期"或"正式"，且角色为hr |

###### 字段权限前端控制

根据 GET /api/v1/employees/field-permissions 接口返回的权限信息动态控制：

```typescript
// 列表页：根据 viewableFields 控制列的显示/隐藏
// 编辑页：根据 editableFields 控制字段的 disabled 状态
// lockedFields 中的字段置灰 + Tooltip 提示
```

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /api/v1/employees | 查询员工列表（含分页、搜索） |
| GET /api/v1/employees/field-permissions | 获取当前角色字段权限 |

### 2.2.2 员工档案详情页

##### UI&交互

- 页面顶部：员工姓名 + 工号 + 在职状态 Tag + 头像占位
- 分区卡片布局：基础信息、个人信息、工作信息、薪资与合同信息
- 每个分区可折叠
- 敏感字段按权限脱敏展示（如身份证号显示 `3301**********1234`）
- 薪资信息分区：HR/财务可查看，部门主管隐藏，普通员工仅看自己

##### 前端逻辑

###### 详情展示字段

**基础信息：**

| 字段 | 可见性 |
| --- | --- |
| 工号 | 所有人 |
| 系统账号 | 本人 + HR |
| 在职状态 | 所有人 |
| 入职日期 | 所有人 |
| 录用类型 | HR + 本人 |
| 创建时间 | HR + 本人 |

**个人信息：**

| 字段 | 可见性 |
| --- | --- |
| 姓名 | 所有人 |
| 性别 | 所有人 |
| 手机号 | HR + 部门主管 + 本人 |
| 邮箱 | HR + 部门主管 + 本人 |
| 身份证号 | HR + 本人（脱敏） |
| 生日 | HR + 部门主管 + 本人 |
| 户籍地址 | HR + 本人 |
| 现居住地址 | HR + 部门主管 + 本人 |

**工作信息：**

| 字段 | 可见性 |
| --- | --- |
| 所属部门 | 所有人 |
| 职位 | 所有人 |
| 职级 | 所有人 |
| 直接汇报人 | HR + 部门主管 + 本人 |
| 工作地点 | 所有人 |
| 入职类型 | HR + 本人 |

**薪资与合同信息：**

| 字段 | 可见性 |
| --- | --- |
| 合同类型 | HR + 本人 |
| 合同到期日 | HR + 本人 |
| 试用期待遇比例 | HR + 本人 |
| 薪资账套 | HR + 财务 |
| 基本工资 | HR + 财务 + 本人 |

###### 操作按钮

| 按钮 | 交互 | 二次确认 | 显示控制 |
| --- | --- | --- | --- |

| 离职 | 跳转离职申请 | N | hr + 在职 |
| 变更历史 | 打开变更历史抽屉 | N | hr |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /api/v1/employees/{id} | 获取员工详情（按角色返回不同字段） |

### 2.2.3 员工档案编辑页

##### UI&交互

- 与详情页相同的分区卡片布局
- 可编辑字段：正常表单控件
- 锁定字段：Input disabled + Tooltip "如需修改XXX请联系HR / 走调岗流程"
- 「保存」按钮提交，仅提交 changedFields

##### 表单字段（按权限动态控制 editable）

参考 2.2.2 详情字段表，服务端通过 fieldPermissions 返回 editableFields / lockedFields。

##### 操作按钮

| 按钮 | 交互 | 二次确认 |
| --- | --- | --- |
| 保存 | 提交 PUT /api/v1/employees/{id} | N |
| 取消 | 返回详情页 | Y，"未保存的更改将丢失" |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| PUT /api/v1/employees/{id} | 更新员工档案字段 |

---

### 2.2.4 变更历史抽屉

##### UI

- 右侧抽屉 / Modal
- 时间线（Timeline）展示变更记录
- 每条：变更字段（中文名）+ 旧值 → 新值 + 变更类型（Tag）+ 操作人 + 时间

##### 数据源

GET /api/v1/employees/{id}/change-logs

## 5.1 风险评估

- 字段权限矩阵复杂，需逐角色验证详情页、编辑页的字段控制逻辑
- 敏感字段脱敏规则需与后端一致（前端展示用，后端为权威）
- 大数据量列表（1000+）需确保搜索和分页性能
