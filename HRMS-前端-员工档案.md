# 2. 详细设计

## 2.1 前端迭代目标

1. 员工列表页：分页表格 + 筛选栏 + 新增/编辑/删除操作
2. 员工详情页：字段分区展示，关联部门/职位名称
3. 员工新增/编辑表单
4. 批量导出 Excel

## 2.2 迭代具体描述

### 2.2.1 员工列表页

##### UI&交互

- 顶部筛选栏（关键词搜索 + 部门下拉 + 职位下拉 + 状态选择）
- 表格展示，支持分页
- 点击姓名进入详情页
- 每行操作列：查看 / 编辑 / 删除

##### 前端逻辑

###### 列表搜索字段

| 字段名称 | 说明 | 输入方式 | 是否必填 | 默认值 | 字段类型 | 提示文案 | 数据源 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 关键词 | 姓名/工号模糊搜索 | Input | N | - | string | 请输入姓名或工号 | - |
| 部门 | 按部门筛选 | TreeSelect | N | 全部 | number | 请选择部门 | `/api/departments/tree` |
| 职位 | 按职位筛选 | Select + 搜索 | N | 全部 | number | 请选择职位 | `/api/positions/list` |
| 状态 | 在职状态 | Select | N | 全部 | number | 请选择状态 | 枚举：在职(1)/试用期(2)/离职(0) |

###### 列表工具栏

| 按钮 | 交互 | 显示控制 |
| --- | --- | --- |
| 新增员工 | 打开新增弹窗/跳转新增页 | 始终显示 |
| 批量导出 | 导出当前筛选结果为 Excel | 始终显示 |

###### 列表展示字段

| 字段名称 | 说明 | 宽度 |
| --- | --- | --- |
| 工号 | `employeeNo` | 120 |
| 姓名 | 点击跳转详情页 | 120 |
| 性别 | 男/女 | 60 |
| 部门 | `departmentName`，关联查询结果 | 120 |
| 职位 | `positionName`，关联查询结果 | 140 |
| 状态 | Tag：在职(绿)/试用期(蓝)/离职(灰) | 80 |
| 入职日期 | `hireDate`，格式化 YYYY-MM-DD | 120 |
| 邮箱 | `email` | 160 |
| 操作 | 查看 / 编辑 / 删除 | 180 |

###### 操作按钮

| 按钮 | 交互 | 二次确认 | 显示控制 |
| --- | --- | --- | --- |
| 查看 | 跳转详情页（或右侧抽屉） | N | 始终 |
| 编辑 | 打开编辑弹窗 / 跳转编辑页 | N | 始终 |
| 删除 | 调用删除接口 | Y，"确定删除该员工吗？" | 始终 |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| `GET /api/employees/list` | 员工分页列表（含筛选） |
| `POST /api/employees/delete` | 删除员工 |
| `GET /api/departments/tree` | 部门树（部门筛选下拉） |
| `GET /api/positions/list` | 职位列表（职位筛选下拉） |

### 2.2.2 员工详情页

##### UI&交互

- 面包屑：首页 > 员工管理 > 员工详情
- 页面头部：姓名 + 工号 + 状态 Tag
- 分区卡片布局，每区可折叠

##### 详情展示字段

**基础信息：**

| 字段 | 字段名 | 说明 |
| --- | --- | --- |
| 工号 | `employeeNo` | 系统自动生成，只读 |
| 姓名 | `employeeName` | |
| 性别 | `gender` | 0=女 1=男（前端转为中文） |
| 状态 | `status` | 0=离职 1=在职 2=试用期 |
| 入职日期 | `hireDate` | |
| 录用类型 | `employmentType` | FULL_TIME/PART_TIME/INTERN |
| 邮箱 | `email` | |

**组织信息：**

| 字段 | 字段名 | 说明 |
| --- | --- | --- |
| 所属部门 | `departmentName` | 接口返回已关联查询 |
| 职位 | `positionName` | 接口返回已关联查询 |
| 部门ID | `departmentId` | |
| 职位ID | `positionId` | |

**个人信息：**

| 字段 | 字段名 | 说明 |
| --- | --- | --- |
| 身份证号 | `idCard` | 脱敏展示：`3301**********1234` |
| 现居住地址 | `currentAddress` | |
| 紧急联系人 | `emergencyContactName` | |
| 紧急联系人电话 | `emergencyContactPhone` | 脱敏展示 |

##### 操作按钮

| 按钮 | 交互 | 显示控制 |
| --- | --- | --- |
| 编辑 | 跳转编辑页 / 打开编辑弹窗 | 始终 |
| 删除 | 二次确认后删除 | 始终 |
| 返回列表 | 回到列表页 | 始终 |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| `GET /api/employees/detail?id={id}` | 获取员工详情 |

### 2.2.3 员工新增/编辑表单

##### 表单字段

| 字段名称 | 说明 | 输入方式 | 是否必填 | 最大长度 | 字段类型 | 提示文案 | 数据源 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 姓名 | 真实姓名 | Input | Y | 32 | string | 请输入员工姓名 | - |
| 性别 | - | Radio | Y | - | number | - | 枚举：男(1)/女(0) |
| 身份证号 | 加密存储 | Input | N | 18 | string | 请输入身份证号 | - |
| 入职日期 | - | DatePicker | Y | - | date | 请选择入职日期 | - |
| 入职类型 | - | Select | N | - | number | 请选择入职类型 | 枚举 |
| 所属部门 | - | TreeSelect | N | - | number | 请选择部门 | `/api/departments/tree` |
| 职位 | - | Select | N | - | number | 请选择职位 | `/api/positions/list` |
| 录用类型 | - | Select | Y | - | string | - | FULL_TIME/PART_TIME/INTERN |
| 邮箱 | - | Input | N | 64 | string | 请输入邮箱 | - |
| 现居住地址 | - | Input | N | 128 | string | 请输入地址 | - |
| 紧急联系人 | - | Input | N | 32 | string | 请输入联系人姓名 | - |
| 紧急联系人电话 | - | Input | N | 11 | string | 请输入联系电话 | - |

**编辑模式差异**：

| 差异项 | 说明 |
| --- | --- |
| 工号 | 新增时自动生成，编辑时只读展示 |
| 姓名 | 新增必填，编辑时选填（不传则不更新） |
| 部门/职位 | 编辑时变更部门/职位可能触发调岗流程 |

##### 操作按钮

| 按钮 | 交互 | 二次确认 |
| --- | --- | --- |
| 保存 | 提交表单 | N |
| 取消 | 返回上一页 | Y，"未保存的更改将丢失" |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| `POST /api/employees/add` | 新增员工 |
| `PUT /api/employees/update` | 更新员工 |
| `POST /api/employees/generate-employee-no` | 生成工号（可选：新增页加载时预生成） |
| `GET /api/departments/tree` | 部门下拉数据源 |
| `GET /api/positions/list` | 职位下拉数据源 |

### 2.2.4 批量导出

##### UI

- 列表页工具栏「导出」按钮
- 导出当前筛选条件下的全部员工数据为 Excel

##### 数据源

```
GET /api/employees/export?keyword=&departmentId=&status=
```

> 后端返回 Excel 文件流，前端触发浏览器下载。

# 5. 其他

## 5.1 风险评估

- 员工列表数据量较大时需确保分页和后端查询性能
- 身份证号、紧急联系人电话等敏感字段前端展示需脱敏
- 部门/职位下拉数据源需在新增/编辑页加载时预加载
