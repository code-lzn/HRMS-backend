# 2. 详细设计

## 2.1 前端迭代目标

1. 审批工作台：待办列表 + 已办列表 + 我发起的（三Tab）
2. 审批详情页：按审批类型动态渲染申请信息 + 审批历史时间线 + 操作区（通过/拒绝/转交）
3. 委托审批：委托设置页（设置/取消委托）
4. 审批通知：站内消息 + 超时标记（红色高亮）

## 2.2 迭代具体描述

### 2.2.1 审批工作台

##### UI&交互

- 顶部三Tab：待办（Badge红点） / 已办 / 我发起的
- 待办列表：左侧审批类型筛选（全部/入职/转正/调岗/离职/请假/补卡/薪资批次/加班），右侧列表
- 超时审批红色高亮 + 超时标签
- 点击某条待办 → 进入审批详情页
- 列表支持按时间倒序

##### 待办列表展示字段

| 字段名称 | 说明 | 宽度 |
| --- | --- | --- |
| 审批类型 | Tag + 颜色：入职(蓝)/转正(绿)/调岗(橙)/离职(红)/请假(青)/补卡(紫)/薪资(金)/加班(灰) | 100 |
| 摘要 | 自动拼接：申请人 - 类型 - 关键信息（如"张三 - 年假 3天"） | 自适应 |
| 申请人 | 姓名 + 部门 | 120 |
| 发起时间 | format YYYY-MM-DD HH:mm | 140 |
| 截止时间 | 48小时倒计时 / 已超时红色 | 120 |
| 操作 | 查看详情按钮 | 80 |

##### 已办列表展示字段

| 字段名称 | 说明 |
| --- | --- |
| 审批类型 | Tag |
| 摘要 | 同上 |
| 申请人 | 姓名 + 部门 |
| 我的操作 | 通过(绿) / 拒绝(红) / 转交(蓝) |
| 操作时间 | format |
| 审批意见 | 超长截断 + Tooltip |

##### 我发起的展示字段

| 字段名称 | 说明 |
| --- | --- |
| 审批类型 | Tag |
| 摘要 | 同上 |
| 当前状态 | 审批中(蓝) / 已通过(绿) / 已拒绝(红) / 已撤回(灰) |
| 当前审批人 | 姓名 |
| 发起时间 | format |
| 操作 | 撤回（仅第一级审批中且发起人） |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /api/v1/approvals/pending | 待办列表 |
| GET /api/v1/approvals/processed | 已办列表 |
| GET /api/v1/approvals/initiated | 我发起的 |
| POST /api/v1/approvals/{instanceId}/withdraw | 撤回 |

### 2.2.2 审批详情页（核心页面）

##### UI 布局

```
┌──────────────────────────────────────────┐
│  ← 返回工作台                             │
│  审批类型 Tag  +  业务标题                 │
├──────────────────────────────────────────┤
│                                          │
│  ┌── 申请信息 ─────────────────────────┐  │
│  │ （按审批类型动态渲染不同字段）        │  │
│  │  入职审批：姓名/性别/手机/部门/职位... │  │
│  │  请假审批：请假类型/时间/天数/事由...   │  │
│  │  调岗审批：原部门→新部门/原因...      │  │
│  │  薪资审批：批次汇总/异常统计...       │  │
│  └──────────────────────────────────────┘  │
│                                          │
│  ┌── 审批历史 ─────────────────────────┐  │
│  │  ○ 第一步：直接上级 — 李四            │  │
│  │  │  通过 · 2024-07-10 10:00          │  │
│  │  │  意见：同意                        │  │
│  │  │                                  │  │
│  │  ● 第二步：部门负责人 — 王五（当前）   │  │
│  │     待审批                            │  │
│  └──────────────────────────────────────┘  │
│                                          │
│  ┌── 审批操作（仅当前审批人可见）────────┐  │
│  │  [通过]  [拒绝]  [转交]              │  │
│  └──────────────────────────────────────┘  │
└──────────────────────────────────────────┘
```

##### 审批类型 → 动态渲染配置

| 审批类型 | 渲染组件 | 数据字段 |
| --- | --- | --- |
| 入职审批 | 入职信息卡片 | 姓名/性别/手机/邮箱/身份证号(脱敏)/入职日期/部门/职位/录用类型/试用期 |
| 转正审批 | 转正评估卡片 | 员工信息/试用期起止/表现评价/薪资调整/审批结果 |
| 调岗审批 | 调岗对比卡片 | 原部门→新部门/原职位→新职位/原职级→新职级/原因 |
| 离职审批 | 离职信息卡片 | 员工信息/离职日期/离职类型/原因/交接人 |
| 请假审批 | 请假信息卡片 | 请假类型/起止时间/天数/事由/交接人/附件 |
| 补卡审批 | 补卡信息卡片 | 补卡日期/补卡类型/补卡时间/原因 |
| 薪资审批 | 薪资批次卡片 | 月份/人数/应发合计/实发合计/异常统计/预览入口 |
| 加班审批 | 加班信息卡片 | 加班日期/起止时间/时长/原因 |

##### 审批历史时间线

- Timeline 组件（AntD）
- 已完成节点：绿色圆点 + 通过/拒绝/转交 Tag
- 当前节点：蓝色圆点 + 闪烁动画
- 未到达节点：灰色圆点

##### 审批操作

| 按钮 | 交互 | 必填项 | 二次确认 |
| --- | --- | --- | --- |
| 通过 | Modal输入意见（可选） | N | Y |
| 拒绝 | Modal输入原因 | Y | Y |
| 转交 | Modal选择转交人 + 原因 | Y | Y |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /api/v1/approvals/{instanceId}/detail | 审批详情 |
| POST /api/v1/approvals/{instanceId}/approve | 通过 |
| POST /api/v1/approvals/{instanceId}/reject | 拒绝 |
| POST /api/v1/approvals/{instanceId}/transfer | 转交 |

### 2.2.3 委托审批设置页

##### UI

- 当前生效委托列表（卡片/表格）
- 「新建委托」弹窗
- 每个委托卡片：委托人 → 被委托人 + 委托时间范围 + 委托类型范围 + 状态（生效中/已取消/已过期）+ 取消按钮

##### 新建委托弹窗

| 字段名称 | 说明 | 输入方式 | 是否必填 | 字段类型 | 数据源 |
| --- | --- | --- | --- | --- | --- |
| 被委托人 | 选择 | Select + 搜索 | Y | number | 员工搜索 |
| 开始日期 | >= 今天 | DatePicker | Y | date | - |
| 结束日期 | > 开始日期 | DatePicker | Y | date | - |
| 委托类型 | 空=全部类型 | Select multiple | N | array | 审批类型枚举 |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| POST /api/v1/approval-delegations | 设置委托 |
| PUT /api/v1/approval-delegations/{id}/cancel | 取消委托 |
| GET /api/v1/approval-delegations | 我的委托列表 |

### 2.2.4 审批流定义管理（管理员功能）

##### UI

- 审批类型列表 + 节点配置展开
- 每个审批类型展示：类型名称 + 节点链（节点1 → 节点2 → [条件节点] → 节点3）
- 编辑节点：选择审批人类型（指定人/角色/部门负责人/直接上级）+ 指定值 + 超时时间 + 条件设置
- 「启用/禁用」开关

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /api/v1/approval-definitions | 审批流定义列表 |
| PUT /api/v1/approval-definitions/{id} | 更新审批流定义 |

# 5. 其他

## 5.1 风险评估

- 8种审批类型的申请信息字段各不相同，需设计好动态渲染配置（建议用 type → component mapping）
- 审批超时红色高亮需前端实时判断（发起时间+48h）
- 委托审批的自动路由对前端透明，但需在审批历史中正确标记
