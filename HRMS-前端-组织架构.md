# 2. 详细设计

## 2.1 前端迭代目标

1. 部门树管理：左侧部门树 + 右侧部门详情，支持新增/编辑/删除/合并
2. 部门人数统计：实时显示各部门在职人数（含子部门递归汇总）
3. 职位管理：职位列表页 + 新增/编辑职位表单
4. 序列职级对照：序列-职级对照表展示

## 2.2 迭代具体描述

### 2.2.1 部门树管理页

##### UI&交互

- 左右分栏布局：左侧部门树（可展开/折叠），右侧部门详情 + 下级部门列表
- 点击树节点切换选中部门，右侧联动更新
- 部门树支持搜索过滤
- 每层节点显示「在职人数」badge

##### 前端逻辑

###### 页面布局

- 面包屑导航：首页 > 组织架构 > 部门管理
- 搜索框：可搜索部门名称，在树中高亮匹配节点
- 工具栏按钮：新增部门、合并部门

###### 部门树组件

| 特性 | 说明 |
| --- | --- |
| 数据加载 | 进入页面一次性加载完整部门树 GET /api/v1/departments/tree |
| 节点展示 | 部门名称 + 在职人数 badge（仅显示含试用期+正式员工数） |
| 展开/折叠 | 默认展开前2级，其他折叠 |
| 选中态 | 蓝色高亮，右侧显示该部门详情 |
| 最大层级 | 5级，第5级的新增按钮隐藏 |
| 空状态 | 显示"暂无部门数据"占位图 |

###### 右侧部门详情面板

展示选中部门的信息：
- 部门名称、部门编码（只读）、上级部门、部门负责人、排序序号、部门描述
- 下级部门列表（表格）：部门名称、编码、负责人、在职人数、排序、操作（编辑/删除）
- 「编辑」按钮 → 弹出编辑弹窗
- 「删除」按钮 → 二次确认 → 校验无子部门+无在职员工 → 删除

###### 新增部门弹窗

| 字段名称 | 说明 | 输入方式 | 是否必填 | 默认值 | 最大输入长度 | 输入限制 | 字段类型 | 提示文案 | 数据源 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 部门名称 | 如"技术部" | Input | Y | - | 32 | - | string | 请输入部门名称 | - |
| 部门编码 | 2位，用于工号 | Input | Y | - | 2 | 字母或数字 | string | 请输入2位编码 | - |
| 上级部门 | 默认为当前选中部门 | TreeSelect | N | 选中部门 | - | - | number | 请选择上级部门 | 部门树 |
| 部门负责人 | - | Select + 搜索 | N | - | - | - | number | 请选择负责人 | GET /api/v1/employees?status=1,2 |
| 排序序号 | 越小越靠前 | InputNumber | Y | 0 | - | 正整数 | number | - | - |
| 部门描述 | 职能说明 | TextArea | N | - | 128 | - | string | 请输入部门描述 | - |

###### 合并部门弹窗

| 字段名称 | 说明 | 输入方式 | 是否必填 | 默认值 | 字段类型 | 提示文案 | 数据源 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 源部门 | 被合并 | Select（置灰当前选中） | Y | - | number | 请选择源部门 | 部门树下拉 |
| 目标部门 | 保留 | Select | Y | - | number | 请选择目标部门 | 部门树下拉 |

校验：
- 源部门 ≠ 目标部门
- 目标部门不能是源部门的子部门
- 合并前展示"将转移 X 名员工和 Y 个子部门，确定合并？"

###### 操作按钮

| 按钮 | 交互 | 二次确认 | 显示/禁用控制 |
| --- | --- | --- | --- |
| 新增部门 | 打开新增弹窗 | N | 层级已达5级时隐藏 |
| 编辑部门 | 打开编辑弹窗 | N | 需选中部门 |
| 删除部门 | 调用删除接口 | Y，"确定删除该部门吗？" | 有子部门/在职员工时禁用 |
| 合并部门 | 打开合并弹窗 | Y，展示转移统计 | 至少2个部门时可用 |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /api/v1/departments/tree | 获取部门树 |
| POST /api/v1/departments | 新增部门 |
| PUT /api/v1/departments/{id} | 编辑部门 |
| DELETE /api/v1/departments/{id} | 删除部门 |
| POST /api/v1/departments/merge | 合并部门 |
| GET /api/v1/employees?status=1,2 | 搜索员工（选负责人） |

### 2.2.2 职位列表页

##### UI&交互

- 顶部筛选栏 + 表格列表
- 序列用 Tab 切换（全部 / M管理 / P专业 / S支持）
- 表格列：职位名称、序列、所属部门、职级范围、默认试用期、操作

##### 前端逻辑

###### 列表搜索字段

| 字段名称 | 说明 | 输入方式 | 是否必填 | 默认值 | 字段类型 | 提示文案 | 数据源 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 序列 | 职位序列 | Tab 切换 | N | 全部 | string | - | 枚举：M/P/S |
| 所属部门 | 按部门过滤 | TreeSelect | N | 全部 | number | 请选择部门 | 部门树 |

###### 列表展示字段

| 字段名称 | 说明 | 默认值 |
| --- | --- | --- |
| 职位名称 | 如"Java开发工程师" | N |
| 序列 | M管理 / P专业 / S支持 | N |
| 所属部门 | 为空显示"全公司通用" | N |
| 职级范围 | 如"P1-P10" | N |
| 默认试用期 | 如"3个月" | N |
| 职位描述 | 超长截断 + Tooltip | N |

###### 列表工具栏

| 按钮 | 交互 | 显示控制 |
| --- | --- | --- |
| 新增职位 | 打开新增弹窗 | 始终显示 |
| 序列职级对照 | 打开对照表抽屉 | 始终显示 |

###### 操作按钮

| 按钮 | 交互 | 二次确认 | 显示控制 |
| --- | --- | --- | --- |
| 编辑 | 打开编辑弹窗 | N | 始终 |
| 删除 | 调用删除接口 | Y，"确定删除？" | 有员工引用时禁用 |

##### 表单字段

| 字段名称 | 说明 | 输入方式 | 是否必填 | 默认值 | 最大输入长度 | 字段类型 | 提示文案 | 数据源 |
