# HRMS-前端-薪资管理

> 实现薪资账套配置、员工薪资档案管理、月度薪资核算全流程（数据准备→计算→预览→审批→发放）、工资条员工自助查看及 AntV 薪资数据可视化。

# 2. 详细设计

## 2.1 前端迭代目标

1. 薪资账套配置：账套CRUD + 工资项目配置（6类项目：固定收入/变动收入/考勤扣款/社保/公积金/个税）
2. 员工薪资档案：薪资档案查询 + 编辑 + 调薪历史
3. 月度薪资核算：批次管理 + 数据准备 + 计算进度 + 预览表格 + 手动调整 + 提交审批
4. 薪资审批：审批列表 + 审批详情
5. 工资条：员工工资条列表 + 详情（二次验证）+ 薪资趋势图
6. AntV 可视化：5张图表（成本趋势/部门分布/构成占比/社保对比/变动分布）

## 2.2 迭代具体描述

### 2.2.1 薪资账套配置页

##### UI&交互

- 左侧账套列表（可切换）+ 右侧工资项目配置
- 账套名称点击切换，选中高亮
- 「新增账套」弹窗，「复制账套」快捷创建
- 工资项目列表支持拖拽排序

##### 账套表单字段

| 字段名称 | 说明 | 输入方式 | 是否必填 | 字段类型 | 数据源 |
| --- | --- | --- | --- | --- | --- |
| 账套名称 | 如"标准职员工资" | Input | Y | string | - |
| 适用范围 | 按部门/职位/职级 | Radio + Select | Y | - | 部门树/职位列表/职级枚举 |
| 适用范围值 | 选中的具体范围 | Select multiple | Y | array | 根据范围类型动态加载 |
| 生效日期 | 账套启用日期 | DatePicker | Y | date | - |

##### 工资项目配置（每个账套内）

| 字段名称 | 说明 | 输入方式 | 是否必填 | 字段类型 | 数据源 |
| --- | --- | --- | --- | --- | --- |
| 项目名称 | 如"基本工资" | Input | Y | string | - |
| 项目类型 | 固定收入/变动收入/考勤扣款/社保扣除/公积金扣除/个税 | Select | Y | number | 枚举 |
| 计算规则 | 公式或规则描述 | Input / TextArea | Y (非个税类) | string | - |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| GET /api/v1/salary-accounts | 账套列表 |
| POST /api/v1/salary-accounts | 新建账套 |
| PUT /api/v1/salary-accounts/{id} | 编辑账套 |
| DELETE /api/v1/salary-accounts/{id} | 删除账套 |
| POST /api/v1/salary-accounts/{id}/items | 添加工资项目 |
| PUT /api/v1/salary-items/{id} | 编辑工资项目 |
| DELETE /api/v1/salary-items/{id} | 删除工资项目 |

### 2.2.3 月度薪资核算页

##### UI&交互（核心页面）

- 顶部：月份选择器 + 批次状态 Timeline（草稿→计算中→待确认→审批中→已通过→已发放）
- 「新建批次」按钮 → 选择月份 → 确认
- 「数据准备」进度条展示：考勤锁定 → 请假拉取 → 加班拉取 → 档案拉取
- 「执行计算」按钮 → 进度条（异步，轮询状态）
- 「预览结果」：大表格展示全员薪资明细

##### 核算步骤状态流

```plain
新建批次 → 数据准备中 → 准备完成 → 计算中 → 计算完成 → 预览确认
                                                          ├── 手动调整（可选）
                                                          ├── 提交审批
                                                          └── 标记异常项
```

##### 预览表格

| 列 | 说明 | 特殊处理 |
| --- | --- | --- |
| 员工工号 | - | - |
| 姓名 | - | - |
| 部门 | - | - |
| 基本工资 | 金额 | 右对齐 |
| 岗位津贴 | 金额 | 右对齐 |
| 绩效奖金 | 金额 | 右对齐 |
| 加班费 | 金额 | 右对齐 |
| 迟到扣款 | 金额（红字） | 红色 |
| 请假扣款 | 金额（红字） | 红色 |
| 应发合计 | 金额加粗 | 底部分类汇总 |
| 社保扣除 | 金额（红字） | 红色 |
| 公积金扣除 | 金额（红字） | 红色 |
| 个税 | 金额（红字） | 红色 |
| 实发合计 | 金额加粗+绿色 | 底部分类汇总 |
| 异常标记 | Tag（黄/红） | Tooltip显示原因 |
| 操作 | 调整 | 打开手动调整弹窗 |

##### 异常行样式

- 黄色预警行：淡黄色背景 + Warning Icon
- 红色预警行：淡红色背景 + Error Icon
- 红色阻断行（无法计算）：整行灰色 + 无法计算提示

##### 手动调整弹窗

| 字段 | 说明 |
| --- | --- |
| 调整金额 | 正数为补发，负数为扣减 |
| 调整原因 | 必填 |
| 影响项 | 选择影响的工资项 |

##### 所需API

| 接口 | 说明 |
| --- | --- |
| POST /api/v1/salary-batches | 新建核算批次 |
| POST /api/v1/salary-batches/{id}/calculate | 执行计算 |
| GET /api/v1/salary-batches/{id}/preview | 预览结果（分页） |
| GET /api/v1/salary-batches/{id}/anomalies | 查看异常项 |
| PUT /api/v1/salary-batches/{id}/adjust | 手动调整 |
| POST /api/v1/salary-batches/{id}/submit | 提交审批 |

### 2.2.5 工资条页（员工视角）

##### 工资条列表

- 按月倒序列表，仅显示已通过/已发放的批次
- 每项：月份 + 应发金额 + 实发金额 + 状态（可查看/不可查看）
- 点击进入工资条详情

##### 工资条详情

- 二次验证弹窗（首次查看）：
  - 选择验证方式：短信验证码 / 登录密码
  - 验证通过后展示详情，10分钟内免验证
- 工资条渲染（仿真实纸质工资条样式）：
  - 表头：月份 + 员工姓名 + 工号 + 部门
  - 收入区域：基本工资/岗位津贴/绩效奖金/加班费 → 小计（应发）
  - 扣除区域：事假扣款/迟到扣款/养老/医疗/失业/公积金/个税 → 小计（应扣）
  - 实发金额：大字号 + 加粗

##### 薪资趋势图（AntV）

- 近6个月实发工资折线图
- 数据源：GET /api/v1/my/salary-trend?months=6

# 5. 其他

## 5.1 风险评估

- 薪资数据高度敏感，前端需配合后端做好二次验证和权限控制
- 月度核算涉及大量数据（500人），预览表格需虚拟滚动 + 分页
- 异常检测逻辑复杂，需确保前端异常标记与后端一致
