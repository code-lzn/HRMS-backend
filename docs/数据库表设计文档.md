7# HRMS 人力资源管理系统 — 数据库表设计文档

> 版本：v1.0 | 日期：2026-07-14 | 数据库：szml | 引擎：InnoDB | 字符集：utf8mb4

## 一、表总览（13张）

| # | 表名 | 模块 | 行数预估 | 说明 |
|---|------|------|----------|------|
| 1 | `user` | 基础 | N | 登录认证 + 5角色权限 |
| 2 | `department` | 组织 | 十~百 | 部门树（自关联） |
| 3 | `position` | 组织 | 十~百 | 职位+职级合一字典 |
| 4 | `employee` | 员工 | N | 员工档案（1:1 扩展 user） |
| 5 | `attendance_record` | 考勤 | N×30/月 | 打卡+请假+加班+外出四合一 |
| 6 | `approval_flow` | 审批 | N×数次/月 | 通用审批引擎（6类业务） |
| 7 | `salary_account` | 薪资 | 十 | 薪资账套模板 |
| 8 | `salary_item` | 薪资 | 百 | 账套下的工资项目+公式 |
| 9 | `employee_salary` | 薪资 | N | 员工薪资档案（基数） |
| 10 | `salary_batch` | 薪资 | 12/年 | 月度核算批次 |
| 11 | `salary_detail` | 薪资 | N×12/年 | 工资条明细 |
| 12 | `salary_change_history` | 薪资 | N×数次/年 | 调薪历史审计 |
| 13 | `income_tax_cumulative` | 薪资 | N×12/年 | 个税累计预扣 |

## 三、逐表详细说明

### 1. user（用户表）

全平台人员基础表。登录认证 + 角色权限，所有 employee_id / create_by / operator_id / approver_id 均关联本表 id。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | BIGINT | ✓ | 主键，雪花算法 |
| `user_account` | VARCHAR(256) | ✓ | 登录账号（工号），唯一索引 |
| `user_password` | VARCHAR(512) | ✓ | 密码，MD5 加盐 |
| `union_id` | VARCHAR(256) | | 微信开放平台 UnionID（预留） |
| `mp_open_id` | VARCHAR(256) | | 公众号 OpenID（预留） |
| `user_name` | VARCHAR(256) | | 用户昵称 / 员工姓名 |
| `user_avatar` | VARCHAR(1024) | | 头像 URL |
| `user_profile` | VARCHAR(512) | | 简介 |
| `user_role` | VARCHAR(256) | ✓ | 角色枚举（见下表） |
| `create_time` | DATETIME | ✓ | 创建时间 |
| `update_time` | DATETIME | ✓ | 更新时间 |
| `is_delete` | TINYINT | ✓ | 逻辑删除：0=正常, 1=删除 |

**角色枚举（user_role）：**

| 值 | 角色 | 数据范围 | 核心权限 |
|----|------|----------|----------|
| `sys_admin` | 系统管理员 | 全平台 | 系统配置、角色管理、数据备份 |
| `hr` | HR 专员 | 全部员工 | 员工管理、薪资核算、考勤、审批 |
| `dept_manager` | 部门主管 | 本部门及下属 | 团队查看、下属审批 |
| `finance` | 财务专员 | 薪资相关 | 薪资审核、成本报表 |
| `employee` | 普通员工 | 仅本人 | 个人信息、请假、工资条 |

### 2. department（部门表）

组织架构树。parent_id 自关联实现树形结构，manager_id 关联部门主管。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | BIGINT | ✓ | 主键，雪花算法 |
| `parent_id` | BIGINT | | 上级部门 ID，NULL = 一级部门 |
| `name` | VARCHAR(128) | ✓ | 部门名称 |
| `manager_id` | BIGINT | | 部门主管 ID → `user.id` |
| `sort_order` | INT | | 排序号 |
| `create_time` | DATETIME | ✓ | 创建时间 |
| `update_time` | DATETIME | ✓ | 更新时间 |

**索引：** `idx_dept_parent`(parent_id), `idx_dept_manager`(manager_id)

**关联：**
- `parent_id` → `department.id`（自关联）
- `manager_id` → `user.id`
- 被 `employee.department_id` 引用
- 被 `attendance_record.department_id` 冗余引用
- 被 `approval_flow.department_id` 冗余引用

### 3. position（职位表）

职位 + 职级合一字典表。`level` 字段兼任职级，减少一张表。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | BIGINT | ✓ | 主键，雪花算法 |
| `name` | VARCHAR(128) | ✓ | 职位名称（唯一索引），如"Java开发工程师" |
| `level` | INT | | 职级：1=初级, 2=中级, 3=高级, 4=资深, 5=专家/经理 |
| `sort_order` | INT | | 排序号 |
| `create_time` | DATETIME | ✓ | 创建时间 |
| `update_time` | DATETIME | ✓ | 更新时间 |

**索引：** `uk_position_name`(name) UNIQUE

**关联：**
- 被 `employee.position_id` 引用
- 被 `salary_account.scope_ids` 引用（适用范围按职位）

| `employee_no` | VARCHAR(32) | ✓ | 工号（唯一索引） |
| `real_name` | VARCHAR(64) | | 真实姓名 |
| `gender` | TINYINT | | 性别：0=女, 1=男 |
| `phone` | VARCHAR(20) | | 手机号 |
| `id_card` | VARCHAR(18) | | 身份证号 |
| `bank_card` | VARCHAR(32) | | 银行卡号 |
| `bank_name` | VARCHAR(64) | | 开户银行 |
| `emergency_contact` | VARCHAR(64) | | 紧急联系人姓名 |
| `emergency_phone` | VARCHAR(20) | | 紧急联系人电话 |
| `department_id` | BIGINT | | 所属部门 → `department.id` |
| `position_id` | BIGINT | | 职位 → `position.id` |
| `employment_status` | TINYINT | | 在职状态（见下表） |
| `hire_date` | DATE | | 入职日期 |
| `probation_end_date` | DATE | | 转正日期 |
| `resign_date` | DATE | | 离职日期 |
| `create_time` | DATETIME | ✓ | 创建时间 |
| `update_time` | DATETIME | ✓ | 更新时间 |

**在职状态枚举（employment_status）：**

| 值 | 状态 | 说明 |
|----|------|------|
| 1 | 试用期 | 入职后、转正前，薪资按试用期比例 |

**索引：**

**字段级权限（以员工档案为例）：**

| 字段 | HR 专员 | 部门主管 | 普通员工 |

| real_name | ✓ 查看 | ✓ 查看 | ✓ 查看 |
| phone | ✓ 查看 | ✓ 查看 | ✓ 查看 |
| id_card | ✓ 查看 | ✗ | ✓（仅自己） |
| bank_card | ✓ 查看 | ✗ | ✓（仅自己） |

| `late_minutes` | INT | | ✓ | | | | 迟到分钟数 |
| `early_minutes` | INT | | ✓ | | | | 早退分钟数 |

- `idx_att_date`(record_date)

### 6. approval_flow（审批流表）

通用审批引擎。`business_type` + `business_id` 关联任意业务，`approval_chain` 定义审批链，`approval_log` 记录每步审批结果。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | BIGINT | ✓ | 主键，雪花算法 |
| `applicant_id` | BIGINT | ✓ | 申请人 ID → `user.id` |
| `department_id` | BIGINT | | 申请人所在部门（冗余）→ `department.id` |
| `business_type` | TINYINT | ✓ | 业务类型（见下表） |
| `business_id` | BIGINT | ✓ | 关联业务表的主键 ID |
| `title` | VARCHAR(256) | ✓ | 审批标题，如"张三-事假申请-3天" |
| `status` | TINYINT | | 0=审批中, 1=已通过, 2=已驳回, 3=已撤回 |
| `current_step` | INT | | 当前审批步骤（1-based） |
| `approval_chain` | TEXT | ✓ | 审批链 JSON（见下方示例） |
| `approval_log` | TEXT | | 审批记录 JSON（见下方示例） |
| `submit_time` | DATETIME | ✓ | 提交时间 |
| `finish_time` | DATETIME | | 审批完成时间 |
| `create_time` | DATETIME | ✓ | 创建时间 |
| `update_time` | DATETIME | ✓ | 更新时间 |

**business_type 枚举：**

| 值 | 业务 | business_id 指向 |
|----|------|------------------|
| 1 | 请假 | `attendance_record.id` |
| 2 | 加班 | `attendance_record.id` |
| 3 | 转正 | `employee.user_id` |
| 4 | 调岗 | `employee.user_id` |
| 5 | 离职 | `employee.user_id` |
| 6 | 薪资审批 | `salary_batch.id` |

**approval_chain JSON 示例：**

```json
[
  { "step": 1, "approver_id": 101, "role": "部门主管" },
  { "step": 2, "approver_id": 102, "role": "HR 专员" },
  { "step": 3, "approver_id": 103, "role": "财务审核" }
]
```

**approval_log JSON 示例：**

```json
[
  { "step": 1, "approver_id": 101, "action": "approve", "comment": "同意", "time": "2026-07-14 10:00:00" },
  { "step": 2, "approver_id": 102, "action": "approve", "comment": "已核实", "time": "2026-07-14 14:30:00" }
]
```

**索引：**
- `idx_approval_applicant`(applicant_id)
- `idx_approval_dept`(department_id)
- `idx_approval_business`(business_type, business_id)
- `idx_approval_status`(status)

薪资计算模板。一个账套 = 一套薪资发放规则，包含多个工资项目。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | BIGINT | ✓ | 主键，雪花算法 |
| `name` | VARCHAR(128) | ✓ | 账套名称，如"标准职员工资" |
| `scope_type` | INT | | 适用范围类型：1=部门, 2=职位, 3=职级 |
| `scope_ids` | VARCHAR(512) | | 适用范围 ID 列表，逗号分隔；NULL=全员 |
| `effective_date` | DATE | | 生效日期 |
| `is_deleted` | TINYINT | | 逻辑删除：0=正常, 1=删除 |
| `create_time` | DATETIME | ✓ | 创建时间 |
| `update_time` | DATETIME | ✓ | 更新时间 |

**关联：**
- 1 : N → `salary_item`（账套包含的工资项目）
- 1 : N → `employee_salary`（使用该账套的员工）

### 8. salary_item（工资项目表）

定义每个账套下的具体工资构成项目。`item_type` 决定使用哪种 Calculator 计算。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | BIGINT | ✓ | 主键，雪花算法 |
| `account_id` | BIGINT | ✓ | 所属账套 → `salary_account.id` |
| `name` | VARCHAR(128) | ✓ | 项目名称，如"基本工资""绩效奖金""养老保险" |
| `item_type` | INT | ✓ | 项目类型（见下表） |
| `formula` | VARCHAR(256) | | 计算公式，如"base*0.08" |
| `sort_order` | INT | | 排序号（工资条展示顺序） |
| `is_taxable` | TINYINT | | 是否计入应纳税所得额：1=是, 0=否 |
| `create_time` | DATETIME | ✓ | 创建时间 |
| `update_time` | DATETIME | ✓ | 更新时间 |

**item_type 枚举 + 对应计算器：**

| 值 | 类型 | Calculator | 示例项目 |
|----|------|------------|----------|
| 1 | 固定收入 | FixedIncomeCalculator | 基本工资、岗位津贴 |
| 2 | 变动收入 | VariableIncomeCalculator | 绩效奖金、加班费 |
| 3 | 考勤扣款 | AttendanceDeductionCalculator | 迟到扣款、请假扣款 |
| 4 | 社保扣除 | SocialSecurityCalculator | 养老 8%、医疗 2%、失业 0.5% |
| 5 | 公积金扣除 | HousingFundCalculator | 公积金 12% |
| 6 | 个税 | IncomeTaxCalculator | 个人所得税（累计预扣法） |

### 10. salary_batch（薪资核算批次表）

每月一次薪资核算的批次记录。走完整状态流转：**草稿 → 计算中 → 待确认 → 审批中 → 已通过 → 已发放**。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | BIGINT | ✓ | 主键，雪花算法 |
| `batch_no` | VARCHAR(32) | ✓ | 批次号，如"SAL202607"（唯一） |
| `salary_month` | VARCHAR(7) | ✓ | 薪资月份 yyyy-MM |
| `status` | INT | | 状态（见下表） |
| `total_employees` | INT | | 参与核算员工数（汇总） |
| `total_gross_pay` | DECIMAL(14,2) | | 应发工资合计（汇总） |
| `total_net_pay` | DECIMAL(14,2) | | 实发工资合计（汇总） |
| `total_tax` | DECIMAL(14,2) | | 个税合计（汇总） |
| `create_by` | BIGINT | | 创建人（HR）→ `user.id` |
| `create_time` | DATETIME | ✓ | 创建时间 |
| `update_time` | DATETIME | ✓ | 更新时间 |

**批次状态流转（status）：**

```
草稿(0) → 计算中(1) → 待确认(2) → 审批中(3) → 已通过(4) → 已发放(5)
                                                 ↘ 已驳回(6) → 修改后重新提交
```

| 值 | 状态 | 可执行操作 |
|----|------|------------|
| 0 | 草稿 | 删除、开始计算 |
| 1 | 计算中 | 等待 |
| 2 | 待确认 | 查看预览、调整、提交审批 |
| 3 | 审批中 | 查看进度 |
| 4 | 已通过 | 工资条可见、发放确认 |
| 5 | 已发放 | 归档 |
| 6 | 已驳回 | 修改后重新提交 |

---

### 11. salary_detail（薪资明细表 / 工资条）

每月核算后每个员工生成一条明细 = **工资条**。`salary_items` 用 JSON 存储各项目金额，避免过多 JOIN。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | BIGINT | ✓ | 主键，雪花算法 |
| `batch_id` | BIGINT | ✓ | 批次 → `salary_batch.id` |
| `employee_id` | BIGINT | ✓ | 员工 → `user.id` |
| `salary_items` | TEXT | | 工资项目明细 JSON |
| `gross_pay` | DECIMAL(12,2) | | 应发工资 |
| `social_security` | DECIMAL(10,2) | | 社保个人部分 |
| `housing_fund` | DECIMAL(10,2) | | 公积金个人部分 |
| `income_tax` | DECIMAL(10,2) | | 个人所得税 |
| `total_deductions` | DECIMAL(10,2) | | 扣款合计 |
| `net_pay` | DECIMAL(12,2) | | **实发工资** = 应发 - 扣款 + 手动调整 |
| `is_abnormal` | TINYINT | | 异常：0=正常, 1=黄色预警, 2=红色预警, 3=红色阻断 |
| `abnormal_reason` | VARCHAR(512) | | 异常原因 |
| `manual_adjustment` | DECIMAL(10,2) | | 手动调整（正=补发, 负=扣回） |
| `adjustment_reason` | VARCHAR(256) | | 调整原因 |
| `payslip_viewed` | TINYINT | | 0=未查看, 1=已验证, 2=已查看 |
| `create_time` | DATETIME | ✓ | 创建时间 |

**salary_items JSON 示例：**

```json
[
  { "name": "基本工资",    "amount": 10000.00, "type": 1 },
  { "name": "岗位津贴",    "amount":  2000.00, "type": 1 },
  { "name": "绩效奖金",    "amount":  3600.00, "type": 2 },
  { "name": "事假扣款",    "amount":  -500.00, "type": 3 },
  { "name": "养老保险",    "amount":  -640.00, "type": 4 },
  { "name": "医疗保险",    "amount":  -160.00, "type": 4 },
  { "name": "失业保险",    "amount":   -40.00, "type": 4 },
  { "name": "住房公积金",  "amount":  -960.00, "type": 5 },
  { "name": "个人所得税",  "amount":  -320.00, "type": 6 }
]
```

**异常检测规则：**

| 条件 | 级别 | is_abnormal |
|------|------|-------------|
| 当月请假 > 15天 | 黄色预警 | 1 |
| 当月加班 > 50小时 | 黄色预警 | 1 |
| 薪资较上月变动 > 30% | 红色预警 | 2 |
| 新入职未设薪资档案 | 红色阻断 | 3 |

**工资计算公式：**
```
应发工资(gross_pay)     = 固定收入 + 变动收入
扣款合计(total_deductions) = 社保 + 公积金 + 个税 + 考勤扣款
实发工资(net_pay)       = 应发工资 - 扣款合计 + 手动调整
```

### 13. income_tax_cumulative（个税累计表）

支持中国个税**累计预扣法**，每员工每月一条。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，雪花算法 |
| `employee_id` | BIGINT | 员工 → `user.id` |
| `tax_year` | INT | 纳税年度，如 2026 |
| `tax_month` | INT | 纳税月份，1~12 |
| `cumulative_gross_pay` | DECIMAL(14,2) | 累计应发工资 |
| `cumulative_threshold` | DECIMAL(14,2) | 累计起征点 = 5000 × 月数 |
| `cumulative_social_security` | DECIMAL(14,2) | 累计社保 |
| `cumulative_housing_fund` | DECIMAL(14,2) | 累计公积金 |
| `cumulative_special_deduction` | DECIMAL(14,2) | 累计专项附加扣除 |
| `cumulative_taxable_income` | DECIMAL(14,2) | 累计应纳税所得额 |
| `tax_rate` | DECIMAL(5,4) | 适用税率 |
| `quick_deduction` | DECIMAL(10,2) | 速算扣除数 |
| `cumulative_tax_payable` | DECIMAL(14,2) | 累计应纳税额 |
| `cumulative_tax_paid` | DECIMAL(14,2) | 累计已缴税额 |
| `current_month_tax` | DECIMAL(10,2) | **本月应缴个税** |
| `create_time` | DATETIME | 创建时间 |

**唯一索引：** `uk_tax_employee_month`(employee_id, tax_year, tax_month)

**计算逻辑：**
```
① 累计应纳税所得额 = 累计收入 - 累计起征点 - 累计社保 - 累计公积金 - 累计专项附加扣除
② 累计应纳税额     = 累计应纳税所得额 × 税率 - 速算扣除数
③ 本月个税         = 累计应纳税额 - 累计已缴税额
```

**个税税率表（综合所得）：**

| 累计应纳税所得额 | 税率 | 速算扣除数 |
|-----------------|------|-----------|
| ≤ 36,000 | 3% | 0 |
| 36,000 ~ 144,000 | 10% | 2,520 |
| 144,000 ~ 300,000 | 20% | 16,920 |
| 300,000 ~ 420,000 | 25% | 31,920 |
| 420,000 ~ 660,000 | 30% | 52,920 |
| 660,000 ~ 960,000 | 35% | 85,920 |
| > 960,000 | 45% | 181,920 |

## 五、设计决策说明

| 决策 | 原因 |
|------|------|
| employee 独立于 user | user 管登录认证，employee 管 HR 敏感字段（身份证、银行卡），字段级权限可控 |
| 考勤+请假+加班四合一 | 避免 3 张表 JOIN，类型字段区分，字段按需使用 |
| 审批流通用化 | 6 类业务共用一个审批引擎，不每种建一张表 |
| 入转调离不建独立表 | 状态存 employee，审批记录即异动轨迹（approval_flow） |
| 工资条用 JSON 存项目 | 避免 salary_detail → salary_item 的多层 JOIN，读取一次即可渲染 |
| 部门 ID 冗余到考勤/审批 | 减少 JOIN，加速部门维度的统计查询 |
