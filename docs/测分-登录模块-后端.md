## 一、测试概述

### 1.1 测试范围

本测分文档覆盖登录模块的全部后端功能，包括：用户注册（参数校验 + 账号唯一性 + 密码加密）、用户登录（密码校验 + Session 写入 + VO 脱敏）、获取当前登录用户（Session 读取 + 数据库二次验证）、用户注销（Session 清除）、权限校验（AOP 切面 + 角色层级模型 + 封号拦截）、用户管理 CRUD（管理员专属）、个人信息修改。

### 1.2 测试目标

- 验证用户注册的完整流程：校验规则正确、账号唯一性保证、密码加密正确
- 验证用户登录的正确性：密码匹配、Session 写入、VO 脱敏字段不泄露
- 验证 Session 认证机制：登录后可以获取用户信息、注销后无法获取
- 验证权限校验的层级模型：高等级角色可通过低等级要求的接口，被拒绝的接口正确返回 403
- 验证被封号用户所有请求被拒绝
- 验证管理员用户管理的 CRUD 完整性
- 验证个人信息修改的安全性：不可通过该接口提权

### 1.3 测试环境

| 项目 | 说明 |
|------|------|
| 数据库 | MySQL 8.0 |
| 测试框架 | JUnit 5 + Mockito + Spring Boot Test |
| API 测试工具 | Postman / REST Assured |
| Session 模拟 | Spring MockHttpSession |
