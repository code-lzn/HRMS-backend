## 1. 测试范围与目标

### 1.1 测试范围

本测分文档覆盖登录模块全部前后端功能，对应 [登录模块-后端系分文档](./后端系分-登录模块.md) 和 [登录模块-前端系分文档](./前端系分-登录模块.md)。

| 模块 | 测试范围 |
|------|----------|
| 用户注册 | 参数校验、账号唯一性、MD5加盐加密、并发控制 |
| 用户登录 | 密码校验、Session写入、VO脱敏、错误提示 |
| 获取当前用户 | Session读取、数据库二次验证、脱敏返回 |
| 用户注销 | Session清除 |
| 权限校验（AOP） | 角色层级模型、@AuthCheck注解拦截、封号拦截 |
| 用户管理（管理员） | 创建/删除/更新/分页查询、ID校验 |
| 个人信息修改 | 提权防御、字段限制 |
| 登录页（前端） | 表单校验、错误提示、记住我、回车提交、Loading防重复 |
| 注册页（前端） | 密码一致性、长度校验、错误提示 |
| 路由守卫 | 未登录拦截、角色权限拦截、redirect回跳 |
| 菜单权限 | 按角色动态渲染 |
| 权限指令 | v-permission:role、v-permission:code、disable模式 |
| Session管理（前端） | 过期处理、刷新恢复、Axios拦截器 |

### 1.2 测试目标

- 验证用户注册完整流程：校验规则正确、账号唯一性保证、密码加密正确
- 验证用户登录正确性：密码匹配、Session写入、VO脱敏字段不泄露
- 验证Session认证机制：登录后可获取用户信息、注销后无法获取
- 验证权限校验层级模型：高等级角色可通过低等级要求的接口，被拒绝接口正确返回403
- 验证被封号用户所有请求被拒绝
- 验证管理员用户管理CRUD完整性
- 验证个人信息修改安全性：不可通过该接口提权
- 验证前端登录页/注册页所有交互和异常状态
- 验证路由守卫正确拦截未登录和无权限用户
- 验证菜单按6种角色正确动态渲染
- 验证权限指令正确控制DOM显隐
