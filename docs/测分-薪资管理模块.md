## 1. 测试范围与目标

### 1.1 测试范围

本测分文档覆盖薪资管理模块全部前后端功能，对应 [薪资管理模块-后端系分文档](./后端系分-薪资管理模块.md) 和 [薪资管理模块-前端系分文档](./前端系分-薪资管理模块.md)。

| 模块 | 测试范围 |
|------|----------|
| 薪资账套管理 | 账套CRUD、工资项目CRUD、拖拽排序、适用范围配置、删除引用保护 |
| 员工薪资档案 | 档案查询/更新、调薪历史自动记录（5种change_type自动识别）、字段级权限 |
| 月度薪资核算 | 批次创建→异步计算→预览→异常检测→手动调整→审批流转→标记发放 |
| 薪资计算引擎 | 策略模式6种计算器、累计预扣法个税、BigDecimal精度、幂等性 |
| 工资条（员工自助） | 列表、二次验证（短信/密码）、详情、个人趋势 |
| 薪资统计 | 5个AntV数据源接口（成本趋势/部门分布/构成占比/社保对比/变动分布） |
| 前端页面 | 账套列表+表单、员工薪资档案详情/编辑、批次列表+轮询、预览页、工资条列表+验证+详情、统计看板 |

### 1.2 测试目标

- 验证账套及工资项目的CRUD完整性、排序正确性、删除引用保护
- 验证员工薪资档案更新后调薪历史自动记录的准确性和change_type自动识别
- 验证异步计算流程正确性（6种计算器 + 累计预扣法个税 + 异常检测 + 明细写入）
- 验证状态机7个状态的流转合法性，非法状态转换被正确拒绝
- 验证幂等性：重跑计算不产生重复数据
- 验证字段级权限（dept_manager看不到薪资字段、user只能看自己）
- 验证个税累计预扣法的计算精度
- 验证工资条二次验证完整流程（短信/密码/失败锁定/30分钟过期）
- 验证前端8个页面所有交互和异常状态
- 验证5张AntV图表正确渲染和联动
