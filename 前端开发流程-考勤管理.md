# 考勤管理模块 - 前端开发流程

## 一、技术栈说明
- **框架**: UmiJS 3.x + React 18
- **组件库**: Ant Design v5
- **构建工具**: Webpack
- **HTTP请求**: Axios (封装于 @/libs/request)

## 四、页面开发详细流程

### 4.1 主页面 index.tsx

**功能**: 整合所有组件，管理页面状态

**状态管理**:
- `filterParams`: 筛选条件（姓名、工号、部门、月份、状态、打卡类型）
- `tableData`: 表格数据
- `pagination`: 分页信息
- `selectedRowKeys`: 批量选中的行ID
- `modalVisible`: 各类弹窗显示状态

**核心逻辑**:
1. 初始化时加载默认当月数据
2. 筛选条件变化时重新查询
3. 分页切换时携带筛选参数
4. 批量操作时校验选中数据

### 4.6 详情弹窗 DetailModal.tsx

**展示内容**:
1. **员工基础信息卡片**
   - 工号、姓名、部门、岗位
2. **当日打卡数据卡片**
   - 打卡日期、上班时间、下班时间、打卡地点
3. **迟到早退加班统计卡片**
   - 迟到时长、早退时长、加班时长
4. **关联审批记录**
   - 请假类型、审批状态

**功能模块**:
1. **模板下载**：调用后端下载接口
2. **文件上传**：限制.xlsx格式
3. **数据预览**：解析Excel后展示表格
4. **错误提示**：标红显示错误数据（重复工号、日期非法、员工不存在）
5. **确认导入**：提交批量数据

**交互逻辑**:
- 文件上传后立即解析并展示预览
- 错误数据标红，禁止导入
- 确认导入后调用后端批量新增接口

### 4.8 删除确认弹窗 DeleteModal.tsx

**交互逻辑**:
- 单条删除：提示"确认删除该考勤记录？"
- 批量删除：提示"确认删除选中的N条考勤记录？"
- 删除后数据无法恢复提示

## 六、API服务封装

### 6.1 services/attendance.ts
```typescript
import request from '@/libs/request';

export async function queryHRAttendance(params: HRAttendanceQueryRequest) {
  return request<API.BaseResponse<PageResult<HRAttendanceVO>>>(
    '/hr/attendance/list',
    { method: 'GET', params }
  );
}

export async function getHRAttendanceDetail(id: number) {
  return request<API.BaseResponse<HRAttendanceVO>>(
    `/hr/attendance/detail/${id}`,
    { method: 'GET' }
  );
}

export async function createHRAttendance(data: HRAttendanceDTO) {
  return request<API.BaseResponse<HRAttendanceVO>>(
    '/hr/attendance/create',
    { method: 'POST', data }
  );
}

export async function updateHRAttendance(data: HRAttendanceDTO) {
  return request<API.BaseResponse<HRAttendanceVO>>(
    '/hr/attendance/update',
    { method: 'POST', data }
  );
}

export async function deleteHRAttendance(id: number) {
  return request<API.BaseResponse<boolean>>(
    `/hr/attendance/delete/${id}`,
    { method: 'DELETE' }
  );
}

export async function batchDeleteHRAttendance(ids: number[]) {
  return request<API.BaseResponse<boolean>>(
    '/hr/attendance/batch-delete',
    { method: 'DELETE', data: ids }
  );
}
```

## 七、权限控制

### 7.1 按钮权限
在页面加载时获取当前用户权限，控制按钮显示/隐藏：
```typescript
const permissions = useSelector(state => state.user.permissions);
const hasPermission = (perm: string) => permissions.includes(perm);
```

### 7.2 数据权限
- 普通员工：只能查看本人考勤（调用员工端接口）
- 部门主管：只能查看本部门员工考勤（后端自动过滤）
- 管理员：无限制（调用HR端接口）

## 八、异常处理

### 8.1 接口错误处理
- 所有接口调用统一使用 try-catch 包裹
- 接口返回错误码时弹出友好提示
- 网络异常时显示重试按钮

### 8.2 无数据处理
- 表格无数据时显示"暂无考勤数据，请切换筛选条件或导入考勤记录"
- 筛选后无匹配数据时显示对应提示

### 8.3 导入错误处理
- Excel格式错误提示
- 数据校验失败标红显示错误原因
- 导入失败时返回详细错误信息

## 九、性能优化

### 9.1 分页加载
- 默认每页10条数据
- 支持切换每页条数（10/20/50/100）

### 9.2 条件缓存
- 筛选条件本地缓存，页面刷新后恢复

### 9.3 懒加载
- 部门下拉框数据懒加载
- 员工选择器搜索时远程加载

## 十、开发注意事项

1. **日期格式**: 统一使用 `YYYY-MM-DD` 格式，时间使用 `HH:mm:ss`
2. **状态映射**: 考勤状态使用枚举映射，确保前后端一致
3. **权限校验**: 所有操作前校验用户权限
4. **表单校验**: 新增/编辑表单必须校验必填字段
5. **批量操作**: 批量操作前必须校验是否有选中数据
6. **loading状态**: 所有接口请求显示loading动画
7. **错误提示**: 所有操作失败必须有清晰的错误提示
8. **代码规范**: 遵循项目已有的代码规范和命名约定
